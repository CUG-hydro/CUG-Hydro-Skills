# 1 编程哲学

## 1.1 核心原则

| 原则 | 含义 |
|------|------|
| **Do one thing well** | 专精一事 |
| **Composition over complexity** | 组合胜于嵌套 |
| **Small is beautiful** | 简洁胜于复杂 |
| **Fail fast, fail loud** | 快速失败，明确报错 |

## 1.2 编码准则

- 遵守极简主义，代码尽可能的简短和简洁，去冗余变量、抽象。**但必须遵守代码排版规范，排版美观**！

- 每行代码不要过长（一般不超过100字符）。

- 避免深层嵌套

- 变量命名要有明确的意义


## 1.3 检查清单

- [ ] 能否删减？
- [ ] 能否组合？
- [ ] 能否简化？
- [ ] 是否早返回？
- [ ] 是否明确失败？
