# 极简主义检查器

按 Linux 极简主义标准审查代码。

## Linux 极简主义原则

源自 Unix 哲学，核心是 **Do one thing well**：

1. **单一职责** - 每个模块只做好一件事
2. **无冗余** - 避免不必要的变量和过度抽象
3. **浅层嵌套** - 嵌套不超过 3 层，超过需重构
4. **早返回** - 用守卫语句在开头处理边界情况，避免 else
5. **行宽控制** - 每行 ≤100 字符
6. **明确错误** - 显式检查，不隐藏失败

## 审查流程

复制此清单并跟踪进度：

```
审查进度：
- [ ] 读取 AGENTS.md 确认规范
- [ ] 检查单一职责
- [ ] 检查冗余变量/抽象
- [ ] 检查嵌套深度
- [ ] 检查行宽
- [ ] 检查排版一致性
- [ ] 检查早返回模式
- [ ] 检查错误处理
```

## 审查清单

| 检查项 | 说明 |
|--------|------|
| 单一职责 | 是否专精一事？ |
| 无冗余 | 是否有不必要的变量/抽象？ |
| 嵌套深度 | ≤3 层？超过需重构 |
| 行宽 | ≤100 字符？ |
| 排版 | 是否美观统一？ |
| 早返回 | 是否在开头使用守卫语句？ |
| 错误处理 | 是否明确清晰？ |

## 输出格式

```
审查结果: [通过 / 需修改]

问题（如有）：
- 行 XX: 具体问题

建议：
1. xxx
2. xxx
```
